# Разработка информационной системы для подбора площадок и командной работы при организации молодёжных мероприятий  

## 1. Общая информация

Организаторам молодёжных мероприятий требуется единая система для поиска подходящих помещений, включая государственные площадки с неполными или недоступными данными. Необходимы инструменты для уточнения информации, формирования и сравнения бюджетов, а также совместной работы команды в рамках одного проекта.

---

# 2. Требования к системе 

## Пользователи
- **Организаторы мероприятий и команды организаторов**  
  Составляют проекты, ищут помещения, формируют бюджеты, совещаются с командой. 
  Участвуют в обсуждении, голосуют, оставляют комментарии, анализируют данные.
- **Представители площадок**  
  Обновляют данные о помещениях, подтверждают доступность, отвечают на запросы.

---

# 3. Функциональные требования

## Клиент для организаторов

### Подбор помещений
- Просмотр каталога государственных и частных площадок.
- Фильтры:
  - вместимость;
  - стоимость;
  - тип пространства;
  - наличие оборудования;
  - доступные даты;
  - район города.
- Просмотр карточки помещения:
  - описание, фото, схема зала;
  - оборудование;
  - правила и ограничения;
  - контактные данные;
  - доступность по датам.

### Сбор информации
- Отправка запросов представителям площадок.
- Добавление вручную недостающих данных (после звонка или посещения).
- Прикрепление файлов (прайсы, схемы, договоры).

### Кластеризация помещений
- Автоматическая группировка по типу, стоимости, размеру, оборудованию и другим доступным данным.
- Ручное создание групп: «бюджетные варианты», «центр города», «большие сцены».
- Сохранение кластеров.

### Работа с бюджетами
- Создание наборов затрат: аренда, техника, персонал, декор, логистика.
- Копирование и редактирование бюджетов.
- Сравнение нескольких вариантов по структуре расходов.

### Командная работа
- Создание команд.
- Приглашение участников по ссылке или email.
- Назначение ролей.
- Совместное обсуждение вариантов.

### Оценки и голосование
- Комментарии к помещениям.
- Голосование: «нравится» / «не подходит».
- Просмотр агрегированных результатов.

### Проект мероприятия
- Создание проекта.
- Добавление возможных площадок и бюджетов.
- Формирование итогового решения.
- Экспорт итогового плана в PDF.

---

## Клиент для представителей площадок

### Управление помещением
- Редактирование описания, фото, оборудования.
- Загрузка документов: правила аренды, прайс, схема зала.

### Доступность
- Указание свободных / занятых дат.

### Работа с запросами
- Просмотр запросов организаторов.
- Ответы: текстом, файлами, уточнениями.

---

# 4. Сервер

- Хранит данные о помещениях, проектах, бюджетах, командах.
- Обрабатывает запросы от мобильных клиентов.
- Обеспечивает обмен уведомлениями:
  - ответы от площадок,
  - комментарии,
  - новые участники,
  - обновления в проекте.
- Контролирует права доступа.

---

# 5. Дополнительный контекст

Система предлагается командой организаторов как удобный инструмент, который:

- упрощает поиск государственных и частных площадок;
- позволяет быстро уточнять информацию у представителей площадок;
- помогает командам принимать коллективные решения;
- снижает расходы благодаря сравнению бюджетов;
- объединяет все процессы подготовки мероприятия в единой системе.
